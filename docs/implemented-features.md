# 実装機能解説

このドキュメントは、`IL Designer Studio` として実装されている画像編集モックの主要機能と内部構造を説明する。対象は `index.html`、`app.js`、`styles.css`、`localize-ja.js` に含まれる現在の実装である。

## 概要

本アプリは、ブラウザ上の Canvas API で動作する Photoshop 風の画像編集 UI である。単一の HTML 画面に、メニューバー、ツールバー、キャンバス、右側パネル群を配置し、JavaScript 側でドキュメント状態、レイヤー、選択範囲、履歴、フィルター、表示補助を管理する。

主な処理は `app.js` に集約されている。`index.html` は操作 UI と Canvas 要素を提供し、`styles.css` はデスクトップ編集ソフトに近いダークテーマのレイアウトを定義する。`localize-ja.js` は画面上の英語ラベルを日本語風のラベルへ置換する後処理として読み込まれる。

## 画面構成

画面は次の領域で構成される。

- メニューバー: File、Edit、Image、Filter、Layer、Select、View の各コマンドを提供する。
- オプションバー: 選択中ツールに応じたサイズ、透明度、許容値、文字、図形、補正値などを操作する。
- 左ツールバー: 移動、選択、ペン、ブラシ、消しゴム、文字、ズームなどのツールを切り替える。
- 中央キャンバス: 編集対象ドキュメントを表示し、ドラッグ操作やポインター操作を受け付ける。
- 右パネル: Navigator、Histogram、Info、Color、Channels、Brush Presets、Properties、Filters、Transform、Layer Style、Layers、Mask、Selection、Paths、History を表示する。

レイアウトは CSS Grid と Flexbox で構成され、右パネルや上部 UI はショートカットで表示切り替えできる。

## ドキュメントと描画モデル

アプリの状態は `state` オブジェクトに集約される。ドキュメントサイズ、レイヤー配列、アクティブレイヤー、選択範囲、表示倍率、パン位置、ブラシ設定、補正値、履歴などを持つ。

描画は主に次の流れで行われる。

1. `composeDocument()` が表示可能なレイヤーを下から合成する。
2. レイヤーマスク、クリッピングマスク、描画モード、塗り、不透明度、Blend If、レイヤースタイル、調整レイヤーを反映する。
3. `render()` が合成結果を表示用 Canvas に描画する。
4. グリッド、ガイド、選択範囲、パス、ブラシカーソル、変形ハンドルなどのオーバーレイを重ねる。

レイヤーの画像データは個別の Canvas として保持される。破壊的な編集は対象レイヤーやマスクの Canvas を直接更新し、必要に応じて履歴へスナップショットを積む。

## ファイル操作

File メニューと上部ボタンから、以下の操作を実行できる。

- 新規ドキュメント作成
- 画像ファイルの読み込み
- 画像ファイルを埋め込み配置
- PNG/JPEG 書き出し
- 初期状態へのリセット

読み込みはブラウザの File API と `createImageBitmap()` を使う。書き出しは合成済み Canvas を画像データ URL として生成し、ダウンロードリンク経由で保存する。

## レイヤー機能

レイヤーは通常レイヤー、背景レイヤー、調整レイヤーを扱う。右側の Layers パネルと Layer メニューから、次の操作ができる。

- 新規レイヤー、複製、削除、リネーム
- レイヤーの上下移動、最前面・最背面への移動
- 表示・非表示、Alt 操作によるソロ表示
- レイヤーの不透明度、塗り、描画モード変更
- Blend If による暗部・明部の透過
- 背景レイヤーと通常レイヤーの変換
- クリッピングマスク
- 透明ピクセル、ピクセル編集、位置、全体のロック
- 下のレイヤーと結合、表示レイヤーを結合、表示状態をスタンプ、画像統合

背景レイヤーは一部の操作が制限される。編集制限は `guardPixelEditing()`、`guardPositionEditing()`、`guardTransparencyEditing()` などで判定される。

## レイヤーマスクとチャンネル

レイヤーマスクは、表示マスクと非表示マスクの追加、無効化、適用、反転、削除に対応する。マスクが存在するレイヤーでは、編集対象を Pixels または Mask に切り替えられる。

選択範囲はアルファチャンネルとして保存できる。Channels パネルでは RGB チャンネルの表示切り替えと、保存済みアルファチャンネルの読み込み・削除を扱う。チャンネル表示はプレビュー用の合成結果に反映される。

## 選択範囲とパス

選択機能は矩形選択、投げ縄、マジックワンド、色域選択、被写体選択、類似色選択、選択範囲の拡張・縮小・ぼかし・境界化に対応する。

選択範囲は矩形またはアルファマスクとして保持される。コピー、カット、削除、塗りつぶし、コンテンツに応じた塗りつぶし、選択範囲内へのペーストなどの編集操作は、この選択マスクを参照して対象ピクセルを限定する。

ペンツールは作業パスを作成する。作成したパスは選択範囲へ変換でき、ブラシ設定を使ったパスのストロークにも対応する。

## 描画ツール

左ツールバーから以下の編集ツールを使用できる。

- Move: レイヤー移動と自動選択
- Marquee、Lasso、Magic Wand: 選択範囲の作成
- Pen: 作業パスの作成
- Brush、Eraser: ブラシ描画と消去
- Spot Healing Brush、Clone Stamp、History Brush: 修復、複製、履歴元からの復元
- Gradient、Shape: グラデーション塗りと図形レイヤー作成
- Smudge、Dodge/Burn: ピクセルのにじませ、覆い焼き、焼き込み
- Type: テキストレイヤー作成
- Eyedropper: 前景色または背景色の取得
- Hand、Zoom、Crop: 表示移動、拡大縮小、切り抜き

ブラシはハード円形、ソフト円形、エアブラシ、四角、カリグラフィのプリセットを持つ。サイズ、不透明度、硬さ、間隔、流量を変更できる。

## 色と補正

Properties パネルと Image メニューから、明るさ、コントラスト、彩度、色相、グレースケール、ぼかしなどの基本補正を扱う。

追加の画像補正として、レベル補正、トーンカーブ、色相・彩度、自然な彩度、カラーバランス、白黒、露光量、シャドウ・ハイライト、自動トーン、自動カラー、自動コントラスト、階調反転、彩度低下、しきい値、ポスタリゼーションが実装されている。

調整レイヤーを作成した場合は、通常のピクセルを直接変更せず、合成時に下位レイヤーへ補正を適用する。

## フィルター

Filter メニューと Filters パネルから、多数のフィルターを適用できる。主な分類は次の通りである。

- ぼかし: Gaussian Blur、Box Blur、Surface Blur、Smart Blur、Radial Blur、Motion Blur
- 変形: Twirl、Pinch、Ripple、Spherize、ZigZag、Wave、Polar Coordinates、Shear、Offset
- シャープ・輪郭: Sharpen、Unsharp Mask、Smart Sharpen、High Pass、Find Edges、Glowing Edges
- 表現効果: Oil Paint、Emboss、Diffuse、Wind、Trace Contour、Extrude、Tiles、Solarize
- 生成・質感: Clouds、Difference Clouds、Lens Flare、Fibers、Lighting Effects
- ノイズ・粒状: Add Noise、Reduce Noise、Despeckle、Dust & Scratches、Median、Mezzotint
- ピクセル化: Color Halftone、Crystallize、Pointillize、Fragment、Facet、Mosaic
- 明暗範囲: Maximum、Minimum

フィルターはアクティブレイヤーのピクセルまたはマスクを対象に適用される。選択範囲がある場合は、ローカルマスクで対象領域を限定する。

## 変形、整列、キャンバス操作

Transform パネルと Image/Layer メニューから、以下の操作を行える。

- 自由変形
- レイヤーまたはキャンバスの 90 度回転、180 度回転
- レイヤーまたはキャンバスの左右・上下反転
- 選択範囲またはキャンバス基準の左右・上下・中央整列
- 画像サイズ変更
- キャンバスサイズ変更
- 透明ピクセルのトリミング
- 全領域表示
- 切り抜き

自由変形と切り抜きは、Enter で確定、Esc でキャンセルするモーダル操作として扱われる。

## 表示補助

View メニューとキャンバス周辺には、編集を補助する表示機能がある。

- Fit on Screen、Actual Pixels、拡大・縮小
- Navigator による表示位置と倍率の操作
- Histogram による合成結果の輝度分布表示
- Info パネルによる座標、RGB、HEX のサンプリング
- ルーラー表示
- グリッド表示
- ガイドの追加、ドラッグ、クリア
- グリッド・ガイドへのスナップ
- Extras、Panels、Tools and Panels の表示切り替え

ガイドはルーラーからドラッグして作成できる。スナップが有効な場合、移動や変形時にグリッドまたはガイドへ吸着する。

## 履歴

履歴は編集操作ごとのスナップショットとして保存される。Undo、Redo、Step Backward、Step Forward に対応する。History パネルでは過去の履歴状態を選択して復元できる。

履歴スナップショット機能も実装されている。任意の時点のサムネイル付きスナップショットを保存し、後で復元できる。History Brush は、このスナップショットをソースとして現在のレイヤーへ部分的に描き戻す。

## ショートカット

キーボード操作は `window.keydown` でまとめて処理される。代表的なショートカットは次の通りである。

- `V`、`M`、`W`、`L`、`P`、`B`、`E`、`T`、`Z` など: ツール切り替え
- `Space`: 一時的な Hand Tool
- `Ctrl+Z`、`Ctrl+Shift+Z`、`Ctrl+Alt+Z`: 履歴操作
- `Ctrl+0`、`Ctrl+1`: 全体表示、等倍表示
- `Ctrl+T`: 自由変形
- `Ctrl+A`、`Ctrl+D`、`Ctrl+Shift+I`: 選択操作
- `Alt+Backspace`、`Ctrl+Backspace`: 前景色・背景色で塗りつぶし
- `Tab`、`Shift+Tab`: UI 表示切り替え
- 数字キー: ブラシまたはレイヤー不透明度の変更
- `Shift+Plus`、`Shift+Minus`: 描画モードの切り替え

## 日本語化処理

`localize-ja.js` は、ページ読み込み後に DOM 内のテキスト、`title`、`aria-label` を走査し、登録済みの英語文言を対応する日本語ラベルへ置換する。さらに `MutationObserver` により、後から追加・変更されたテキストにも同じ置換を適用する。

この方式は既存の HTML とアプリ本体を大きく変更せずに表示文言だけを置換できる一方で、辞書にない文言は英語のまま残る。また、現在の辞書には文字化けしたように見える文字列が含まれているため、実運用で正しい日本語表示が必要な場合は辞書データの文字コードまたは内容を修正する必要がある。

## 実装上の注意点

- アプリはフロントエンドのみで完結し、サーバー保存や PSD の実ファイル互換は実装していない。
- レイヤー、履歴、選択範囲はメモリ上に保持されるため、大きな画像や長時間の編集ではメモリ使用量が増える。
- 多くの画像処理は Canvas のピクセル配列を直接走査するため、画像サイズやフィルター種類によって処理時間が増える。
- `app.js` は機能が集中した単一ファイルであり、関数間の状態共有は `state` と DOM 参照に依存している。
- UI はデスクトップ幅を前提に作られており、小さい画面では右パネルやメニューが非表示になる。

## 確認観点

機能を確認する場合は、次の順で見ると全体像を把握しやすい。

1. 画像を開く、または初期サンプルレイヤーを表示する。
2. レイヤーの追加、移動、表示切り替え、不透明度変更を試す。
3. 選択範囲を作成し、塗りつぶし、コピー、削除、フィルター適用を試す。
4. マスクを追加し、編集対象を Mask に切り替えてブラシで編集する。
5. フィルター、補正、調整レイヤー、レイヤースタイルを適用する。
6. Undo/Redo と History Snapshot で状態が復元できることを確認する。
7. PNG または JPEG として書き出す。
